Mission Briefing

How to operate OverEats

OverEats is a food-delivery platform presented as a military-style logistics control panel. The theme is cosmetic; the workflow is normal: accounts, restaurants, menu items, orders, deliveries, notes, and LiveTrack updates.

Use the steps below to create realistic data manually.

Role Clearance

Command Matrix

CustomerPlaces orders, creates notes
DriverViews assigned active deliveries

Operational Steps

Manual Setup

01 — Establish Identity Access

  1. Open Identity Access.
  2. Register one restaurant operator, one customer operator, and optionally one driver operator.
  3. Use saved operator profiles to switch roles quickly.

02 — Deploy Restaurant Assets

  1. Switch to the restaurant operator.
  2. Create a restaurant asset from the dashboard.
  3. Add at least one menu asset to that restaurant.

03 — Submit Order Operation

  1. Switch to the customer operator.
  2. Use the restaurant ID and menu item ID from the snapshot cards.
  3. Submit the order with normal special instructions.

04 — Assign Field Delivery

  1. Switch back to the restaurant operator.
  2. Enter the order ID and driver user ID.
  3. Assign the field unit. The resulting delivery ID can be used in LiveTrack.

05 — Operate LiveTrack Signals

  1. Use the delivery ID in the LiveTrack panel.
  2. Send chat messages, GPS updates, and delivery status updates.
  3. The bridge uses the active operator password for LiveTrack authentication.

06 — Archive Notes

  1. Switch to the customer operator who placed the order.
  2. Create private order notes from the Archive & Notes panel.
  3. Retrieve, export, or restore note blobs from the same panel.

Dashboard Terms

Translation Layer

Mission Console

Shows the raw API response from the latest button action. Use it to see IDs, errors, and LiveTrack output.

Pizza Index

Inspired by the Pentagon Pizza index, the pizza score calculated in the browser from session state, tracked IDs, events, and service health.

Tracked Mission IDs

The latest restaurant, menu item, order, and delivery IDs. These are copied into related forms when possible.

Recent Activity

A local browser feed of actions taken from the GUI. It is not server-side logging.
