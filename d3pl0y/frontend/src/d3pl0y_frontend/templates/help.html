API & VM Reference

Everything the web UI does is also available over a small HTTP API.

Authentication

Register a username to receive a secret token. Authenticate API requests with HTTP Basic auth — username as the user and your token as the password.

Authorization: Basic base64("<username>:<token>")

Object API — /user/<owner>/<objectname>

MethodAction
GETDownload the object's bytes.
PUTCreate or overwrite the object (request body = bytes).
DELETEDelete the object.
POSTRun an action (JSON body), one of:
 
{"action": "execute", "argument": "<str>"}
{"action": "make_public"}
{"action": "set_permission", "scope": "exact|prefix",
 "receiver": "<username>|*public", "permission": "r|w|x", "effect": "allow|deny"}

Examples

curl -u "alice:$TOKEN" -X PUT --data-binary @prog.bin http://HOST:2553/user/alice/hello
curl -u "alice:$TOKEN" http://HOST:2553/user/alice/hello
curl -u "alice:$TOKEN" -H 'Content-Type: application/json' \
     -d '{"action":"execute","argument":"world"}' http://HOST:2553/user/alice/hello

Access control

An object is (owner, name); name may contain / to form prefix folders, but storage is flat. Owners always have full r/w/x on their own objects. Every grant is one rule:

  • scope — exact (one object) or prefix (a folder; the prefix must end in /).
  • receiver — a username, or *public for everyone.
  • permission — read / write / execute.
  • effect — allow or deny.

Resolution: the most specific rule wins (longest matching prefix); at equal specificity, deny beats allow.
